Creating dashboard packs

  • Step 1 - create some awesome dashboards
  • Step 2 - link your dashboards
    Ensure that your Overview dashboard contains markdown tiles that contain links to sub-dashboards. Don't worry about the IDs themselves, just that they match; the Configurator will swap them later. Example: "markdown": "## [üîç](#dashboard;id=38862cbd-8ab4-4e17-81ca-1f43af427ad3)" needs to match: "id": "38862cbd-8ab4-4e17-81ca-1f43af427ad3",
  • Step 3 - determine your swaps
    The Configurator will look for strings to swap. They should be in the format: ${MyString} See below for a predefined list. If you need new swaps, include them in the template. By default, the user will get a prompt for these (coming soon). If you need custom handling, e.g. a pulldown instead of a text input or need choices from the API, note that in your request to add the dashboard pack, and be sure to include example API queries to get the choices.
  • Step 4 - test locally
    Click the gear icon in the top right and add your repo and Overview dashboard. Test a deployment.
  • Step 5 - submit you dashboardpack
    Either submit your pack on our forum or email the maintainers directly.
